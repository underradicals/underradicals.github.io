What is a Program ​

Languages ​

What is a Program? It is a collection of instructions, often written in text, but not necessarily written in text, and to a varying degree may or may not resemble something the machine understands. The more alien the program is to a machine we say the language the program is comprised of is a higher level language and the more familiar a language is to a machine the language is of a lower level.

A program ​

This is an assembly program that sums the numbers 1-10 and prints out the sum.

asm
section .data
    msg db "Sum = ", 0
    msg_len equ $ - msg

    newline db 10     ; newline character

    ; buffer to store string version of result (max 3 digits + newline + null)
    buffer db "000", 10, 0

section .bss
    sum resd 1

section .text
    global _start

_start:
    ; initialize registers
    mov ecx, 1      ; counter = 1
    xor eax, eax    ; sum = 0

sum_loop:
    add eax, ecx    ; sum += counter
    inc ecx         ; counter++
    cmp ecx, 10     ; while counter <= 10
    jle sum_loop

    mov [sum], eax  ; store result

    ; Convert integer in EAX to ASCII (buffered)
    mov ebx, eax    ; copy sum to ebx
    mov edi, buffer
    add edi, 2      ; start at third digit from left
    mov ecx, 3      ; 3 digits max (since 1+2+...+10 = 55)

convert_loop:
    xor edx, edx
    mov eax, ebx
    mov ebx, 10
    div ebx         ; EAX = quotient, EDX = remainder
    add dl, '0'     ; convert remainder to ASCII
    mov [edi], dl
    dec edi
    mov ebx, eax
    loop convert_loop

    ; Write "Sum = "
    mov eax, 4          ; sys_write
    mov ebx, 1          ; stdout
    mov ecx, msg
    mov edx, msg_len
    int 0x80

    ; Write the number
    mov eax, 4
    mov ebx, 1
    mov ecx, buffer
    mov edx, 4          ; 3 digits + newline
    int 0x80

    ; Exit
    mov eax, 1          ; sys_exit
    xor ebx, ebx
    int 0x80
c
#include <stdio.h>

int main() {
    int sum = 0;

    for (int i = 1; i <= 10; ++i) {
        sum += i;
    }

    printf("Sum = %d\n", sum);
    return 0;
}
python
print(f"Sum = {sum(range(1, 11))}")

Not so simple ​

A simple program in todays standards, yet the complexity involved in writing the program is injective, we can't fit all of that in our brains. However, look at the difference between the three languages: asm, c and python respectively each one more alien to the language a machine would speak. From asm to c we have almost a single order of magnitude lest text. From c to python we have exactly one order of magnitude less text to write. And from asm to python, yep you guessed it, nearly two orders of magnitude less text. Less text means less time to complete and less time means programs are more accessible, more affordable, which means more programs will be written. And boy have they!

What is a Language ​

But what is a language? Well, that can get pretty technical. Languages like English or Japanese are called natural. However, a language like C or Java are called Formal Languages A language has many parts:

  • Vocabulary (Alphabet)
  • Grammar
  • Syntax
  • Rules

    A Formal Language does not have semantics like it has a Grammar or Vocabulary. The semantics are usually constructed in the compiler or transpiler; in either case semantical meaning of the language is existential to the language.

These different parts of the language have nuances of their own: For instance, a finite word over an vocabulary is any finite sequence of letters. A vocabulary can be thought of as a subset of some maybe larger non-empty set whose members are all finite-length words. This carries over to other disciplines like formal logic where words are called formulas and/or sentences. The Grammar determines how the words of the vocabulary are formed or concatenated. Concatenation is only one of the operations used to validate a word in a vocabulary: You also have, an intersection, complement, reversal and string homomorphism. Now this technical after this point, don't worry we have plenty of time to jump in the deep end later. I am not a genius, at that point we will be learning together.

Complexity ​

Together these things determine what the language can do, how it can do them, and more importantly it determines exclusions or what it cannot do. It also, and from a developers perspective, more importantly it determines how much mental effort it takes to solve a problem. Because, even though a language is meant as a bridge between people and machines, it can very often become a stumbling stone to the developer, baying him/her/them from the problem at hand. And that after all is the job, solving real world problems. If a language makes that harder either through syntax gymnastics, or poorly constrained language features the problem often becomes buried under a pile of mental complexity; this results in the problem being solved poorly or even worse, not at all.

To Machine Code ​

After a program has been written and tested it has to be turned into something the machine can understand. You computer does not understand C, Java, or Python. So in order for the machine to execute the instructions you spent weeks perfecting it needs to transformed into machine code. But we can't just turn text into machine code outright, that is not how it works, as usual things are more complex than that. Lets take our C program from before.

sum.c ​
c
#include <stdio.h>

int main() {
    int sum = 0;

    for (int i = 1; i <= 10; ++i) {
        sum += i;
    }

    printf("Sum = %d\n", sum);
    return 0;
}

In order to turn this into machine code it must endure steps:

  • Preprocessing
  • Compilation
  • Assembly
  • Linking

Now we can get the gcc compiler to be explicit when it turns sum.c into sum.exe by providing some flags along with the command.

bash
gcc -v --save-temps -o sum main.c

The --save-temps make the compiler output a file for each intermediary step in the compilation process. We will get a list of files:

  • sum-main.i
  • sum-main.s
  • sum-main.o
  • sum.exe
